# 队列操作性能测试计划

## 目标
修改C、C++、Rust、Go、C#和Java六种语言的测试程序，实现队列操作性能测试：
1. 创建长度为1000的字符型队列
2. 插入随机字符（A-Z,a-z,0-9）
3. 队列满时自动出队，保持队列长度不超过1000
4. 循环10000次，计算总耗时

## 实施步骤

### 1. 队列实现策略
- **C**: 使用数组模拟队列，维护front和rear指针
- **C++**: 使用STL的std::queue，结合size()检查
- **Rust**: 使用std::collections::VecDeque
- **Go**: 使用container/list或自定义切片
- **C#**: 使用System.Collections.Generic.Queue
- **Java**: 使用java.util.LinkedList

### 2. 随机字符生成
所有语言生成范围：A-Z（65-90）、a-z（97-122）、0-9（48-57），共62种可能

### 3. 程序结构
```
初始化队列
记录开始时间
循环10000次：
    生成随机字符
    入队
    如果队列长度 > 1000：
        出队
记录结束时间
计算并输出总耗时（毫秒）
```

### 4. 具体修改方案

**C语言**
- 使用数组实现循环队列
- 维护front和rear指针
- 使用rand()生成随机数

**C++语言**
- 使用std::queue<char>
- 使用rand()或C++11随机库

**Rust语言**
- 使用VecDeque<char>
- 使用rand crate生成随机数

**Go语言**
- 使用container/list
- 使用math/rand生成随机数

**C#语言**
- 使用Queue<char>
- 使用Random类生成随机数

**Java语言**
- 使用LinkedList<Character>
- 使用java.util.Random生成随机数

### 5. 注意事项
- 确保所有语言的队列操作逻辑一致
- 随机字符生成方式一致
- 循环次数严格为10000次
- 时间测量方式保持不变（毫秒）

### 6. 执行顺序
1. 修改C程序
2. 修改C++程序
3. 修改Rust程序
4. 修改Go程序
5. 修改C#程序
6. 修改Java程序
7. 运行测试脚本验证结果

## 预期结果
通过运行修改后的测试程序，可以得到六种语言执行相同队列操作的性能对比，反映它们在数据结构操作上的效率差异。
